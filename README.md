## 🖥️ What is vmtrace?

vmtrace is a simple tool that helps you watch and track virtual machines running on Windows. It uses the Windows Hypervisor Platform to run guest systems and record what they do. This can be useful if you want to understand more about how these virtual machines work or if you need to check their performance.

You do not need to know coding or technical details to use this tool. It is made for anyone who wants to run and inspect a virtual machine safely on a Windows computer.

## ⚙️ System Requirements

Before you use vmtrace, make sure your PC meets these conditions:

- Windows 10 version 1809 or newer, or Windows 11
- A 64-bit processor with virtualization support turned on in BIOS (Intel VT-x or AMD-V)
- At least 8 GB of RAM (16 GB recommended for better performance)
- 500 MB free disk space for the program and its files
- Administrator privileges to run the tool with full access

If you are unsure if your CPU supports virtualization, you can check this by typing "System Information" in your Start menu and looking for "Virtualization Enabled in Firmware".

## 🔍 How to Use vmtrace

Once vmtrace is running, you will see a simple window with options.

### 1. Set up a new virtual machine

- Click on "Create new VM."
- Choose an operating system you want to run inside the virtual machine.
- Select how much memory and CPU power the VM will use. Use the slider or type in numbers.
- Click "Start VM" to launch it.

### 2. Trace the VM activity

- When the VM is running, vmtrace records its actions.
- You can see logs or reports inside the main window.
- Use tabs to switch between memory use, CPU instructions, and system calls.

### 3. Stop and save your work

- Click "Stop VM" to pause or end the session.
- You can save the trace data for later analysis by clicking "Save Trace."

This data helps you understand what happened inside the VM during its run.

## 🛠️ Troubleshooting

If vmtrace does not start or runs slowly, try the following fixes:

- Check if your virtualization is enabled in BIOS or UEFI settings.
- Run vmtrace as administrator. Right-click the executable and select "Run as administrator."
- Close other heavy programs to free up CPU and memory.
- Restart your computer if you encounter errors.

If your VM fails to launch, make sure you chose a supported operating system in the setup window.

## 🔒 Security Tips

Running virtual machines carries some risk if you visit unsafe websites inside them.

- Use vmtrace mainly for testing or learning.
- Avoid downloading files inside the VM unless you trust the source.
- Always keep your Windows system updated to protect from threats.
